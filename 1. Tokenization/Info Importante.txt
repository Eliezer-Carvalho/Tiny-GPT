1. Pre tokenizer (define como o texto é segmentado antes do modelo de aprendizagem)
2. Algoritmo (Algoritmo de Aprendizagem)


Alguns Pontos Importantes nesta fase:

vocab_size é uma variável importante porque influencia o número final de tokens, por norma é importante estudar esta variável.
special_tokens é também importante porque define como o algoritmo de aprendizagem vai reagir em certas ocasiões

O tokenizer é treinado num dataset para aprender um vocabulário adaptado
Após isto temos os Tokens e temos os ID's dos tokens


Tokenização é uma forma de compressão estatística do texto
menos tokens = mais eficiência
mais tokens = mais detalhe


Outras fases importantes não referidas aqui:
1. Normalization - Nem sempre é obrigatório, depende do objetivo
2. Falar tokenizer, ids, encode, decode
3. Esta fase não é obrigatória, 90% usa-se um Tokenizer geral


block size = quantos tokens o modelo vê numa sequencia (32 - 8000+)
batch_size = quantas sequencias vê ao mesmo tempo (4 - milhares)

Embedding transforma cada token num vetor de números contínuos
Embedding transforma “símbolos” em algo que o modelo consegue entender
Palavras semelhantes ficam com vetores parecidos
Transformers funcionam com multiplicações, somas e produto escalar e isso só faz sentido com vetores, não com IDs
